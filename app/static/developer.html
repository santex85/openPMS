Getting started

  1. Register and obtain credentials — POST /auth/register creates a tenant and owner user; POST /auth/login returns JWT access and refresh tokens.
  2. First authenticated request — call any scoped route with header Authorization: Bearer <access_token>, or use an API key from POST /api-keys with header X-API-Key: openpms_....
  3. Webhooks — create a subscription with POST /webhooks/subscriptions; events are delivered as POST to your HTTPS URL with header X-Webhook-Signature: sha256=... (HMAC over raw body).

Authentication

User sessions use short-lived access JWTs and refresh tokens (see /auth/login, /auth/refresh). Integrations should use API keys with granular scopes (see table below). All tenant-scoped routes require a resolved tenant from the token or key.

Regional settings (country packs)

Built-in regional profiles (country packs) define currency, timezone, default times, and tax rules. Flow:

  1. GET /country-packs — list available packs.
  2. GET /country-packs/{code} — full definition (taxes, payment methods).
  3. POST /country-packs/{code}/apply with body {"property_id": "<uuid>"} — applies the pack to a property (may be blocked after bookings exist; see GET /properties/{property_id}/lock-status).

Folio lines can include auto-calculated taxes from the active pack; integrators (e.g. TM30) often attach extensions under /country-packs/extensions and property activation endpoints.
